# 超采集算系统 V0.1 工程介绍

## 一、项目概述

### 1.1 项目背景

超采集算系统是一个**高性能数据采集与算力破解一体化平台**，采用"树莓派主控 + Pico2协处理 + Pico终端采集"的三级分布式架构，实现了64路模拟信号高精度采样与128路数字信号高速捕获，并集成了密码暴力破解功能。

### 1.2 项目定位

- **类型**：嵌入式数据采集系统
- **平台**：树莓派4B + RP2040/RP2350
- **应用场景**：信号采集分析、协议破解、硬件安全测试
- **版本**：V0.1（原型验证版）

### 1.3 核心指标

| 指标类别 | 参数 | 数值 |
|----------|------|------|
| **模拟采样** | 通道数 | 64路 |
| | 分辨率 | 12bit |
| | 单路采样率 | 125KSPS |
| | 总吞吐率 | 8MSPS |
| **数字捕获** | 通道数 | 128路 |
| | 单路捕获率 | 100MSPS |
| | 总吞吐率 | 12.8GSPS |
| **算力破解** | 默认速度 | 50万次/秒 |
| | 超频速度 | 120万次/秒 |
| **系统规模** | Pico芯片 | 16片 |
| | Pico2芯片 | 2片 |
| | 主控 | 树莓派4B 4GB |

## 二、系统架构

### 2.1 总体架构

系统采用**三级分布式架构**，从上到下分别为：

```
┌──────────────────────────────────────────────┐
│              树莓派4B（主控层）                │
│  ┌────────────────────────────────────────┐  │
│  │  Flask Web服务 + SocketIO实时推送       │  │
│  └────────────────────────────────────────┘  │
│  ┌──────────────┐  ┌──────────────────────┐  │
│  │ 业务逻辑进程  │  │ 数据接收/存储进程     │  │
│  └──────────────┘  └──────────────────────┘  │
└─────────────┬────────────────────────┬───────┘
              │ USB CDC (100Mbps)      │
     ┌────────┴───────┐        ┌───────┴────────┐
     │  Pico2 #0      │        │  Pico2 #1      │
     │ (协处理器0)    │        │ (协处理器1)    │
     │ 240MHz超频     │        │ 240MHz超频     │
     └──┬──┬──┬──┬───┘        └──┬──┬──┬──┬───┘
        │  │  │  │  SPI(20Mbps)  │  │  │  │
     ┌──┘  │  │  └──┐         ┌──┘  │  │  └──┐
     ▼     ▼  ▼     ▼         ▼     ▼  ▼     ▼
   Pico  Pico ... Pico      Pico  Pico ... Pico
   #0    #1      #7         #8    #9     #15
   (终端采集芯片，每片200MHz超频)
```

### 2.2 层级职责

| 层级 | 设备 | 主要职责 |
|------|------|----------|
| **主控层** | 树莓派4B | 全局调度、业务逻辑、数据解析、FFT分析、破解运算、Web服务、存储管理 |
| **协处理层** | Pico2 ×2 | 数据聚合、SPI主站管理、USB通信、超频控制 |
| **采集层** | Pico ×16 | ADC采样、数字捕获、数据预处理、SPI从机通信 |

### 2.3 软件架构

树莓派端采用**多进程架构**，各进程独立运行、通过队列通信：

```
┌──────────────────────────────────────────────────┐
│                 系统守护进程 (-20, CPU0)           │
│     进程监控、硬件看门狗、异常恢复                  │
└──────────────────────────────────────────────────┘
┌─────────────┬────────────────────────────────────┐
│ 数据接收进程 │  模式调度进程  故障处理进程          │
│   (-10)     │    (-5)          (-5)    (CPU1)     │
├─────────────┼────────────────────────────────────┤
│ 业务逻辑进程 │  人机交互进程     (CPU2)            │
│    (0)      │     (0)                            │
├─────────────┼────────────────────────────────────┤
│ 存储与远程进程         (10, CPU3)                  │
│ 数据落盘、日志记录、Web服务、预警                   │
└──────────────────────────────────────────────────┘
```

## 三、核心功能模块

### 3.1 模拟采样模块

**位置**：Pico终端芯片 + 树莓派业务逻辑

**功能描述**：
- 16片Pico，每片负责4路ADC采样（共64路）
- 12位分辨率，单路最高125KSPS采样率
- 支持DMA连续采样模式
- 内置温度传感器监测

**关键文件**：
- [adc_driver.h](file:///e:/rp/pico/firmware/adc_driver.h) - ADC驱动头文件
- [adc_driver.c](file:///e:/rp/pico/firmware/adc_driver.c) - ADC驱动实现

**技术特点**：
- 使用ADC Round-Robin模式轮询采样
- DMA自动搬运数据，减少CPU占用
- 数据格式：4bit通道号 + 12bit采样值
- 支持采样率动态调节（1K~125KSPS）

### 3.2 数字捕获模块

**位置**：Pico终端芯片

**功能描述**：
- 16片Pico，每片负责8路数字捕获（共128路）
- 单路最高100MSPS捕获速率
- 支持多种触发模式（上升沿、下降沿、双边沿、电平）
- PIO编程实现高速采样

**关键文件**：
- [digital_capture.h](file:///e:/rp/pico/firmware/digital_capture.h) - 数字捕获头文件
- [digital_capture.c](file:///e:/rp/pico/firmware/digital_capture.c) - 数字捕获实现

**技术特点**：
- PIO状态机实现硬件级捕获，精度达纳秒级
- 支持预触发/后触发深度配置
- 位图数据存储，每字节8通道状态

### 3.3 数据聚合模块

**位置**：Pico2协处理器

**功能描述**：
- 每片Pico2管理8片Pico终端
- 实时轮询采集数据并聚合
- 数据校验、去重、时间戳对齐
- 聚合后通过USB CDC发送到树莓派

**关键文件**：
- [data_aggregate.h](file:///e:/rp/pico2/firmware/data_aggregate.h) - 数据聚合头文件
- [data_aggregate.c](file:///e:/rp/pico2/firmware/data_aggregate.c) - 数据聚合实现

**聚合策略**：
- 时间窗口聚合（默认100ms）
- 缓冲区满触发聚合
- 按节点ID有序排列

### 3.4 超频控制模块

**位置**：Pico/Pico2

**功能描述**：
- Pico：默认133MHz，可超频至200MHz
- Pico2：默认150MHz，可超频至240MHz
- 支持三种模式：默认、高性能、自动
- 自动模式根据温度动态调整频率

**关键文件**：
- [overclock.h](file:///e:/rp/pico2/firmware/overclock.h) - 超频控制头文件
- [overclock.c](file:///e:/rp/pico2/firmware/overclock.c) - 超频控制实现

**安全机制**：
- 超频前提升核心电压
- 温度监控，过热自动降频
- 稳定性检测机制

### 3.5 破解引擎模块

**位置**：树莓派业务逻辑

**功能描述**：
- 支持MD5哈希暴力破解
- 可配置密钥长度（8-32位）
- 可配置字符集
- 实时进度反馈

**关键文件**：
- [business_logic.py](file:///e:/rp/raspberry_pi/business_logic.py) - 业务逻辑模块

**性能参数**：
| 模式 | 速度 | 16位密钥平均耗时 |
|------|------|-----------------|
| 默认 | 50万次/秒 | 18ms |
| 超频 | 120万次/秒 | 7.2ms |

### 3.6 FFT分析模块

**位置**：树莓派业务逻辑

**功能描述**：
- 实时FFT频谱分析
- 主频自动检测

## 四、通信协议

### 4.1 协议层级

```
Web前端 ←WebSocket/HTTP→ 树莓派 ←USB CDC→ Pico2 ←SPI→ Pico
```

### 4.2 Pico ↔ Pico2 (SPI)

**速率**：20Mbps

**帧格式**：
```
┌──────┬──────┬──────────┬───────┬──────┐
│ 帧头  │ 长度  │   数据    │ CRC16 │ 帧尾  │
│ 1B   │ 2B   │  N字节    │  2B   │ 1B   │
│ 0xAA │      │ (最大1KB) │       │ 0x55 │
└──────┴──────┴──────────┴───────┴──────┘
```

**命令集**：
| 命令码 | 功能 |
|--------|------|
| 0x01 | 获取状态 |
| 0x02 | 启动采样 |
| 0x03 | 停止采样 |
| 0x04 | 设置采样率 |
| 0x05 | 获取数据 |
| 0x08 | 超频控制 |

详细协议见 [protocol.md](file:///e:/rp/docs/protocol.md)

### 4.3 Pico2 ↔ 树莓派 (USB CDC)

**速率**：100Mbps

**帧格式**：
```
┌──────┬───────┬──────┬──────────┬───────┬──────┐
│ 帧头  │ 节点ID │ 长度  │   数据    │ CRC32 │ 帧尾  │
│ 1B   │  1B   │ 2B   │  N字节    │  4B   │ 1B   │
│ 0x55 │       │      │ (最大8KB) │       │ 0xAA │
└──────┴───────┴──────┴──────────┴───────┴──────┘
```

### 4.4 树莓派 ↔ Web前端 (WebSocket)

**事件列表**：
| 事件 | 方向 | 说明 |
|------|------|------|
| `status_update` | 服务端→客户端 | 状态更新（100ms/次） |
| `data_update` | 服务端→客户端 | 数据更新（波形、进度） |
| `start_sample` | 客户端→服务端 | 启动采样 |
| `stop_sample` | 客户端→服务端 | 停止采样 |
| `start_crack` | 客户端→服务端 | 启动破解 |
| `set_overclock` | 客户端→服务端 | 设置超频模式 |

## 十、故障排查

| 故障现象 | 可能原因 | 解决方法 |
|----------|----------|----------|
| 开机无反应 | 电池没电/电源管理故障 | 充电30分钟，检查IP5328P |
| OLED不显示 | I2C接线松动/驱动异常 | 重新插拔OLED，检查SDA/SCL |
| 采集无数据 | SPI通信故障/Pico掉线 | 检查SPI接线，重启Pico |
| 超频过热降频 | 风道堵塞/导热垫脱落 | 清理散热孔，重新粘贴导热垫 |
| Web无法访问 | 服务未启动/网络不通 | 检查IP，重启Web服务 |
